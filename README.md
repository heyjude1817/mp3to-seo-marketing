# MP3to.cc SEO Marketing

这个仓库用于集中管理 MP3to.cc 的 SEO 营销资产，包括关键词地图、内容规划、工具页 brief、博客 brief、发布检查清单和外链机会清单。

目标不是替代主项目代码仓库，而是作为内容生产和 SEO 决策的单一工作区：

- 统一沉淀关键词、搜索意图、页面类型和内部链接策略。
- 为每个工具页提供可复用的标题、描述、FAQ、结构化数据和内容模块 brief。
- 为博客、指南、对比页和问题排查内容提供生产计划。
- 跟踪发布前检查项，降低多语言、canonical、schema 和内部链接遗漏。

## 项目背景

MP3to.cc 是一个浏览器端在线工具站，核心卖点是免费、快速、隐私优先、无需上传文件。主站覆盖音频转换、视频转音频、音频处理、图片工具、字幕工具和硬件测试工具。

SEO 的主战场应聚焦在高意图工具词和可解释性强的教育内容：

- 工具词：`mp3 to wav converter`、`wav to mp3 converter`、`mp4 to mp3 converter`
- 问题词：`how to convert mp3 to wav`、`why convert wav to mp3`
- 对比词：`wav vs mp3`、`flac vs mp3`、`aac vs mp3`
- 工作流词：`batch audio conversion`、`audio compression for podcast`

## 使用方式

1. 先从 `KEYWORD-MAP.md` 选择目标关键词。
2. 根据页面类型在 `briefs/tools` 或 `briefs/blog` 创建 brief。
3. 发布前使用 `playbooks/publishing-checklist.md` 做检查。
4. 发布后把内部链接和外链机会记录到 `assets/link-opportunities.md`。

## 内容原则

- 每个页面只服务一个主搜索意图。
- 工具页优先解决转换任务，内容解释服务于完成任务，而不是堆文字。
- 博客页承担教育、对比、排错和工作流场景。
- 内部链接必须围绕真实下一步：转换、压缩、剪切、合并、格式选择。
- 所有营销主张必须可被产品体验支撑，避免无法验证的性能承诺。
